Descripción detallada

Clase central del subistema gráfico.

Se encarga de gestionar todas las entidades que se pintan en la ventana de la aplicación.

Está implementado como un Singleton de inicialización explícita; se debe llamar a su método Init(), para que la instancia esté creada. Al final de la aplicación, habrá que llamar a Release().

Los detalles de dibujado están implementados haciendo uso de Ogre.

CScene * Graphics::CServer::createScene ( const std::string &  name)

Crea una nueva escena.

Puede ser de un estado como el menú o un nivel del juego. Su equivalente en la lógica del juego en el caso de ser un nivel sería el mapa. la escena es un contenedor de todas las entidades gráficas que pueden ser reenderizadas por un viewport asociado a esa escena.

Comentarios
El servidor gráfico SE HACE RESPONSABLE de la liberación de la escena al terminar la aplicación. No obstante se puede liberar antes mediante removeScene(). Si el motor gráfico se destruye antes del objeto que creó la escena, este objeto dejará de tener la escena accesible.
En depuración se garantiza que no se creen escenas con el mismo nombre.
Parámetros
nameNombre que se le quiere dar a la escena. En depuración se garantiza que no se creen escenas con el mismo nombre.
Devuelve
Escena creada.

bool Graphics::CServer::Init ( )
static

Inicializa el servidor gráfico.

Dependiendo de la configuración (release o debug), utiliza pantalla completa o no.

Por conveniencia, el Init crea la ventana minimizada, por lo que la aplicación después deberá llamar a ShowWindow(). Se hace así para que la aplicación no muestre una ventana en negro mientras dura todo el proceso de carga. La ventana está creada y el usuario la verá, pero al menos la ve minimizada.

Devuelve
Devuelve false si no se ha podido inicializar.
